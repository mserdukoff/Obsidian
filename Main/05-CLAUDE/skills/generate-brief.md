# Skill: Generate Brief

## Trigger phrases
"generate a brief for [topic]" / "brief this connection" / "make a brief"

## Process
Create a content brief with exactly these five sections:

### ONE THING
The single insight this piece is built around.
Must be one sentence. If it cannot be stated in one sentence, the idea
is not ready. Push back and ask Matt to sharpen it before proceeding.

### PROOF
The most specific real example, number, or linguistic example that
proves the one thing. If writing about grammar: show the actual sentence
in the target language, not a description of it. Vague proof invalidates
the brief.

### READER TRANSFORMATION
What does the reader know or feel at the end that they did not before?
If this cannot be stated clearly, the piece has no reason to exist.

### THREE HOOKS (ranked)
Rank by scroll-stopping power.
Hook 1 — aggressive: confronts the reader with something uncomfortable
Hook 2 — curious: opens a question they want answered
Hook 3 — personal: makes it about their specific situation

### THREE CLOSERS (ranked)
Rank by urgency and memorability.
The closer is written before the middle. Always.

## Output
Save as a new note in 03-BRIEFS/ with filename:
YYYY-MM-DD [topic].md
Tag it #ready-to-write and the relevant project tag.

## Type check
Before writing the brief, identify output type:
- TYPE 1 (short-form social): hooks should be ≤ 12 words
- TYPE 2 (long-form/SEO): hooks can be full sentences, closer is a CTA
If unclear, ask Matt which type before proceeding.
