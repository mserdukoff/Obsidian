# Skill: Write Content

## Trigger phrases
"write the brief for [topic]" / "write this brief" / "produce content from brief"

## Process
1. Read the specified brief from 03-BRIEFS/
2. Read all source notes linked in the brief
3. Identify the output type (TYPE 1 or TYPE 2) from the brief
4. Write the full piece in Matt's exact voice as defined in CLAUDE.md
5. Structure: hook → proof → body → closer
6. Every sentence must earn its place. No filler, no throat-clearing.
7. If writing about grammar or language: always show the actual sentence
   in the target language. Never describe a grammar rule without
   demonstrating it with a real example.

## Voice checklist before finalizing
- [ ] Sentences are short and declarative
- [ ] The sharpest point comes first, not third
- [ ] Every claim has a specific example or number attached
- [ ] None of the banned words appear: "delve," "dive into,"
      "game-changer," "unlock," "leverage," "comprehensive,"
      "it's worth noting," "in conclusion"
- [ ] Reads like Matt wrote it, not like AI wrote it

## TYPE 1 output (short-form social)
- Hook stands alone on the first line
- No more than 280 words for Twitter/X
- TikTok script: write as spoken lines, one idea per line,
  include a [SHOW THIS] note wherever a grammar example
  should appear on screen

## TYPE 2 output (long-form/SEO)
- Include a suggested title tag and meta description at the top
- Use H2s for major sections, no H3s unless absolutely necessary
- End with a CTA that naturally references Grammario if relevant

## Output
Save draft in 03-BRIEFS/ next to the source brief.
Filename: YYYY-MM-DD [topic]-draft.md
Tag it #written
