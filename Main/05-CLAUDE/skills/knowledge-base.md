# Skill: Knowledge Base

## Trigger phrases
"add to knowledge base" / "connect this concept" / "knowledge session"
"link these ideas" / "what do I know about [topic]"

## Philosophy
This is not a second inbox. The knowledge base is a living map of
what Matt actually understands — about building products, engineering,
distribution, and anything else that accumulates over time.

Notes here are connected by concept, not by project. An insight about
user retention from Grammario belongs next to a note about retention
from any other product Matt studies — not siloed inside a Grammario folder.

## Where knowledge lives
All knowledge base notes live in 01-CAPTURES/patterns/ with the tag
#knowledge and a concept tag (e.g. #retention #distribution #pricing
#systems #devtools #growth #architecture #ux).

## Adding a new concept note

### Format for every knowledge note:
---
tags: [#knowledge, #concept-name, #project-tag-if-relevant]
related: [[note-1]], [[note-2]]
date: YYYY-MM-DD
---

# [Concept name]

## What it is
One to three sentences. Plain language. No jargon unless the jargon
is the point.

## Why it matters for building
One to two sentences. Concrete, not abstract.

## Where I've seen it
Bullet list of specific examples from Matt's own notes or experience.
Real instances only — no generic textbook examples.

## Connected concepts
Links to other knowledge notes this one talks to.
The connection must be stated explicitly:
"This relates to [[pricing psychology]] because both depend on..."

## Open questions
Things Matt doesn't know yet about this concept. These automatically
become candidates for 01-CAPTURES/questions/.
